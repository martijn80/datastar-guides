# Datastar attribute modifiers and tags Cheatsheet

This overview shows all Datastars attribute modifiers and which tags can be used on what attributes. 
This is a sort of inverted [Attributes cheatsheet](attributes-cheatsheet.md).

| modifier      | tags                                                                                                                                                                                                                                                                                                                                                                                                        | Can be applied to attribute(s)                                                                                                                                                                             |
|--------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `__case`     | Converts the casing of the key / event / signal name.<br><br>.camel – Camel case, e.g. `mySignal` / `myEvent`.<br>.kebab – Kebab case, e.g. `my-signal` / `my-event`.<br>.snake – Snake case, e.g. `my_signal`.<br>.pascal – Pascal case, e.g. `MySignal` / `MyEvent`.                                                                                                                                                            | `data-bind`, `data-class`, `data-computed`, `data-indicator`, `data-on`, `data-ref`, `data-signals`                                                                                              |
| `__self`     | Only ignore the element itself, not its descendants, when Datastar walks the DOM.                                                                                                                                                                                                                                                                                                                                               | `data-ignore`                                                                                                                                                                                     |
| `__delay`    | Delays execution of the listener/expression by a fixed time.<br><br>.500ms – Delay for 500 milliseconds (accepts any integer before `ms`).<br>.1s – Delay for 1 second (accepts any integer before `s`).                                                                                                                                                                                                                          | `data-init`, `data-on`, `data-on-intersect`, `data-on-signal-patch`                                                                                                                              |
| `__viewtransition` | Wraps the expression in `document.startViewTransition()` when the View Transition API is available, enabling smooth view transitions around that update.                                                                                                                                                                                                                                                                 | `data-init`, `data-on`, `data-on-intersect`, `data-on-interval`                                                                                                                                   |
| `__terse`    | Outputs JSON in a more compact format without extra whitespace, useful for inline display when debugging signals.                                                                                                                                                                                                                                                                                                                | `data-json-signals`                                                                                                                                                                               |
| `__once`     | Runs the listener only once, then removes it.<br>For `data-on`, only supported for built-in browser events (not arbitrary custom events).                                                                                                                                                                                                                                                                                        | `data-on`, `data-on-intersect`                                                                                                                                                                    |
| `__passive`  | Marks the event listener as passive, meaning it will not call `preventDefault` and the browser can optimize scrolling, etc.                                                                                                                                                                                                                                                                                                     | `data-on`                                                                                                                                                                                         |
| `__capture`  | Attaches the listener in the capture phase instead of the default bubbling phase.                                                                                                                                                                                                                                                                                                                                               | `data-on`                                                                                                                                                                                         |
| `__debounce` | Debounces the listener so it only runs after events have stopped firing for a given time.<br><br>.500ms – Debounce for 500 milliseconds (accepts any integer).<br>.1s – Debounce for 1 second (accepts any integer).<br>.leading – Use leading edge: run once immediately, then debounce (must come after timing tag).<br>.notrailing – Disable trailing edge: don’t run at the end of the debounce period (must come after timing tag). | `data-on`, `data-on-intersect`, `data-on-signal-patch`, `data-on-resize`                                                                                                                         |
| `__throttle` | Throttles how often the listener can run, at most once per duration window.<br><br>.500ms – Throttle to once every 500ms (accepts any integer).<br>.1s – Throttle to once every 1s (accepts any integer).<br>.noleading – Don’t run on leading edge; first call is delayed until after the first interval (must come after timing tag).<br>.trailing – Ensure a trailing call when the throttle window ends (must come after timing tag).     | `data-on`, `data-on-intersect`, `data-on-signal-patch`, `data-on-raf`, `data-on-resize`                                                                                                          |
| `__window`   | Attaches the event listener to the global `window` object instead of the element.                                                                                                                                                                                                                                                                                                                                               | `data-on`                                                                                                                                                                                         |
| `__outside`  | Triggers the handler only when the event occurs outside the element (useful for click-outside behavior).                                                                                                                                                                                                                                                                                                                        | `data-on`                                                                                                                                                                                         |
| `__prevent`  | Calls `evt.preventDefault()` on the event before running the expression.                                                                                                                                                                                                                                                                                                                                                        | `data-on`                                                                                                                                                                                         |
| `__stop`     | Calls `evt.stopPropagation()` on the event before running the expression.                                                                                                                                                                                                                                                                                                                                                       | `data-on`                                                                                                                                                                                         |
| `__half`     | For intersection observers: trigger when half of the element is visible in the viewport.                                                                                                                                                                                                                                                                                                                                        | `data-on-intersect`                                                                                                                                                                               |
| `__full`     | For intersection observers: trigger when the whole element is visible in the viewport.                                                                                                                                                                                                                                                                                                                                          | `data-on-intersect`                                                                                                                                                                               |
| `__duration` | Sets the duration for interval-based events.<br><br>.500ms – Interval duration of 500 milliseconds (accepts any integer).<br>.1s – Interval duration of 1 second (default).<br>.leading – Execute the first interval immediately, instead of waiting for the first tick.                                                                                                                                                           | `data-on-interval`                                                                                                                                                                                |
| `__ifmissing`| Only patches signals whose keys do not already exist. Useful for setting defaults without overwriting existing values.                                                                                                                                                                                                                                                                                                          | `data-signals`                                                                                                                                                                                    |
| `__session`  | Uses `sessionStorage` instead of `localStorage` as the persistence target for signals.                                                                                                                                                                                                                                                                                                                                           | `data-persist`                                                                                                                                                                                    |
| `__filter`   | Filters out empty values when syncing signal values to query-string params, so only non-empty values appear in the URL.                                                                                                                                                                                                                                                                                                         | `data-query-string`                                                                                                                                                                               |
| `__history`  | Enables history support: each time a matching signal changes, a new browser history entry is pushed, and signal values are restored from the query string on back/forward navigation.                                                                                                                                                                                                                                            | `data-query-string`                                                                                                                                                                               |
| `__smooth`   | Scrolls the element into view using smooth animated scrolling.                                                                                                                                                                                                                                                                                                                                                                   | `data-scroll-into-view`                                                                                                                                                                           |
| `__instant`  | Scrolls the element into view instantly, without animation.                                                                                                                                                                                                                                                                                                                                                                     | `data-scroll-into-view`                                                                                                                                                                           |
| `__auto`     | Lets the browser choose scroll behavior based on the computed `scroll-behavior` CSS property.                                                                                                                                                                                                                                                                                                                                   | `data-scroll-into-view`                                                                                                                                                                           |
| `__hstart`   | Aligns so the element’s left edge is at the start of the horizontal scroll viewport.                                                                                                                                                                                                                                                                                                                                            | `data-scroll-into-view`                                                                                                                                                                           |
| `__hcenter`  | Aligns the element to the horizontal center of the scroll viewport.                                                                                                                                                                                                                                                                                                                                                             | `data-scroll-into-view`                                                                                                                                                                           |
| `__hend`     | Aligns so the element’s right edge is at the end of the horizontal scroll viewport.                                                                                                                                                                                                                                                                                                                                             | `data-scroll-into-view`                                                                                                                                                                           |
| `__hnearest` | Scrolls to the nearest horizontal edge of the element (start or end) relative to the current viewport.                                                                                                                                                                                                                                                                                                                          | `data-scroll-into-view`                                                                                                                                                                           |
| `__vstart`   | Aligns so the element’s top edge is at the start of the vertical scroll viewport.                                                                                                                                                                                                                                                                                                                                               | `data-scroll-into-view`                                                                                                                                                                           |
| `__vcenter`  | Aligns the element to the vertical center of the scroll viewport.                                                                                                                                                                                                                                                                                                                                                               | `data-scroll-into-view`                                                                                                                                                                           |
| `__vend`     | Aligns so the element’s bottom edge is at the end of the vertical scroll viewport.                                                                                                                                                                                                                                                                                                                                              | `data-scroll-into-view`                                                                                                                                                                           |
| `__vnearest` | Scrolls to the nearest vertical edge of the element (top or bottom) relative to the current viewport.                                                                                                                                                                                                                                                                                                                           | `data-scroll-into-view`                                                                                                                                                                           |
| `__focus`    | After scrolling into view, calls `.focus()` on the element.                                                                                                                                                                                                                                                                                                                                                                     | `data-scroll-into-view`                                                                                                                                                                           |
